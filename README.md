# Automated Feature Extraction via Sparse Autoencoders for Monosemanticity

## Overview

This project explores Mechanistic Interpretability in Large Language Models by analyzing and disentangling hidden representations of TinyLlama using Sparse Autoencoders (SAEs).

The goal is to transform dense hidden activations into interpretable monosemantic features, analyze their behavior, and investigate whether manipulating specific features can influence model outputs.

---

## Problem Statement

Neural networks often operate in a phenomenon known as superposition, where a single neuron represents multiple concepts simultaneously. This makes model reasoning difficult to interpret and audit.

This project uses Sparse Autoencoders to decompose TinyLlama hidden layer activations into sparse and interpretable latent features that can be analyzed and steered.

---

## Objectives

- Extract hidden activations from TinyLlama.
- Train a Sparse Autoencoder on internal representations.
- Learn sparse latent features representing model concepts.
- Analyze feature activation patterns.
- Perform feature steering by modifying latent representations.
- Explore monosemanticity and model interpretability.

---

## Methodology

### 1. Prompt Generation
Generated 2000 prompts across multiple domains:
- Coding
- Mathematics
- Science
- History
- Logical Reasoning

### 2. Activation Extraction
Collected hidden state activations from Layer 12 of TinyLlama-1.1B-Chat.

### 3. Sparse Autoencoder Training
Trained an SAE to compress and reconstruct hidden activations while enforcing sparsity.

### 4. Feature Analysis
Analyzed learned latent features using:
- Activation frequency
- Feature strength
- Domain-specific behavior

### 5. Feature Steering
Modified selected latent features and injected reconstructed activations back into the model to study behavioral changes.

## Dataset Files

The following files were intentionally excluded from GitHub:

- activations.pt
- features.pt
- sae.pt

Reason:

These files contain generated model activations, learned feature representations, and trained model weights. They are large binary artifacts produced during experimentation and can be regenerated by running the notebook.

Keeping them outside the repository:
- Reduces repository size
- Improves cloning speed
- Follows GitHub best practices
- Avoids storing large model artifacts in source control
